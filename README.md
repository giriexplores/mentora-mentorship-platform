# Mentora Mentorship Platform API

A REST API connecting **parents**, **mentors**, and **students** on a mentorship platform. Built with Express, TypeScript, Drizzle ORM (Neon Postgres), and Google Gemini for AI-powered text summarisation.

> **Live deployment:** https://mentora-mentorship-platform.onrender.com
> **Interactive API docs (Swagger UI):** https://mentora-mentorship-platform.onrender.com/docs/

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Node.js + TypeScript |
| Framework | Express |
| ORM | Drizzle ORM |
| Database | Neon (serverless Postgres) |
| Validation | Zod v4 |
| Auth | JWT (jsonwebtoken) – 7-day expiry |
| Password hashing | bcrypt (12 rounds) |
| LLM | Google Gemini (`gemini-3-flash-preview`) via `@google/genai` |
| API docs | Swagger UI — [live](https://mentora-mentorship-platform.onrender.com/docs/) / local at `/docs` |

## Getting Started

### Prerequisites

- Node.js ≥ 18
- A [Neon](https://neon.tech) Postgres database
- A [Google AI Studio](https://aistudio.google.com/) API key for Gemini

### Environment Variables

Create a `.env` file in the project root. All variables are validated at startup — the server will print a clear error and exit if any are missing.

```dotenv
NODE_ENV=development

# Neon Postgres connection string
DATABASE_URL=postgresql://<user>:<password>@<host>/<db>?sslmode=require

# Google Gemini API key (see section below)
GEMINI_API_KEY=<your-key-here>

# Model name — change to any Gemini model you have access to
LLM_MODEL=gemini-3-flash-preview

# HS256 JWT signing secret — use a long random string in production
JWT_SECRET=super_secret_secret

# Port the HTTP server listens on
PORT=3000
```

### Setting the Gemini API Key

1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey) and sign in with a Google account.
2. Click **Create API key** and copy the generated key.
3. Add it to your `.env` file:

```dotenv
GEMINI_API_KEY=AIza...yourKeyHere
```

The default model is `gemini-3-flash-preview`. You can switch to any model listed at [ai.google.dev/gemini-api/docs/models](https://ai.google.dev/gemini-api/docs/models) by updating `LLM_MODEL`.

### Install & Run

```bash
# Install dependencies
npm install

# Generate SQL migrations based on your Drizzle schema
npm run db:generate

# Apply SQL migrations generated by npm run db:generate
npm run db:migrate

# Start in development mode (auto-restarts via nodemon)
npm run dev

# Build and start in production mode
npm run build && npm start
```

The server starts on `http://localhost:3000`.

Interactive API docs (local): http://localhost:3000/docs

Interactive API docs (live): https://mentora-mentorship-platform.onrender.com/docs/

## API Overview

| Method | Path | Auth | Role | Description |
|--------|------|------|------|-------------|
| `POST` | `/api/v1/auth/signup` | — | — | Register as parent or mentor |
| `POST` | `/api/v1/auth/login` | — | — | Login, receive JWT |
| `GET` | `/api/v1/auth/me` | JWT | any | Get current user profile |
| `POST` | `/api/v1/students` | JWT | parent | Create a student under the parent |
| `GET` | `/api/v1/students` | JWT | parent | List the parent's students |
| `POST` | `/api/v1/lessons` | JWT | mentor | Create a lesson |
| `GET` | `/api/v1/lessons/:id/sessions` | JWT | any | List sessions for a lesson |
| `POST` | `/api/v1/bookings` | JWT | parent | Book a lesson for a student |
| `POST` | `/api/v1/sessions` | JWT | mentor | Record a session for a lesson |
| `POST` | `/api/v1/llm/summarize` | — | — | Summarise text with Gemini |

All authenticated routes expect:

```
Authorization: Bearer <jwt>
```

## Testing the LLM Summarise Endpoint

### Example curl Request

```bash
curl -X POST http://localhost:3000/api/v1/llm/summarize \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Artificial intelligence is transforming industries worldwide. From healthcare to finance, AI models are automating complex tasks, improving decision-making, and enabling personalised experiences at scale. Machine learning algorithms can now detect diseases from medical images with accuracy rivalling that of specialist physicians. In finance, AI-driven fraud detection systems analyse millions of transactions per second. However, these advances come with significant challenges around bias, transparency, and data privacy that the industry must address proactively."
  }'
```

### Example Response

```json
{
  "success": true,
  "data": {
    "summary": "• AI is transforming healthcare, finance, and other industries at scale.\n• ML models match specialist doctors in medical image diagnosis.\n• Finance uses AI to detect fraud across millions of transactions per second.\n• Challenges remain around bias, transparency, and data privacy.",
    "model": "gemini-3-flash-preview"
  }
}
```

### End-to-End Auth Flow (curl)

```bash
# 1. Register a mentor
curl -X POST http://localhost:3000/api/v1/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"mentor@example.com","password":"secret123","confirmPassword":"secret123","role":"mentor"}'

# 2. Log in and grab the token
TOKEN=$(curl -s -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"mentor@example.com","password":"secret123"}' | jq -r '.data.token')

# 3. Create a lesson
curl -X POST http://localhost:3000/api/v1/lessons \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"Introduction to Algebra","description":"Covers basic algebraic expressions"}'
```

## Assumptions & Design Choices

### LLM / Summarisation

| Item | Detail |
|------|--------|
| **Minimum text length** | 50 characters — shorter inputs don't produce meaningful summaries |
| **Maximum text length** | 10 000 characters — avoids excessive token usage and keeps response times reasonable |
| **Output format** | 3–6 bullet points, capped at ~120 words total |
| **Model** | `gemini-3-flash-preview` — fast and cost-effective; configurable via `LLM_MODEL` env var |
| **Authentication** | Not required — the endpoint is intentionally public for ease of testing |
| **Rate limiting** | 10 requests per IP per minute (enforced via `express-rate-limit`); returns HTTP 429 on breach |
| **Error handling** | Returns HTTP 500 with `{ success: false, message: "Failed to generate summary" }` if the Gemini API call fails or returns an empty response |

### Authentication & Roles

| Item | Detail |
|------|--------|
| **JWT expiry** | 7 days — suitable for a mentorship platform where frequent re-authentication would be disruptive |
| **Password hashing** | bcrypt with 12 salt rounds (~250 ms on modern hardware, sufficient brute-force resistance) |
| **Student registration** | Students cannot self-register; they are created by a parent and inherit `role: "student"` |
| **Role enforcement** | Checked per-route via the `requireRole` middleware after JWT verification |

### Data

| Item | Detail |
|------|--------|
| **UUIDs** | All primary keys are UUIDs generated by Postgres (`defaultRandom()`) |
| **Date format** | Session `date` must be a valid ISO 8601 UTC string (e.g. `2026-03-15T10:00:00.000Z`) |
| **Duplicate emails** | Rejected with HTTP 409 across all user types (parent, mentor, student share the same `users` table) |
| **Password visibility** | Passwords are never returned by any endpoint; the `password` column is excluded from all public queries |

### Logging

All inbound HTTP requests are logged to stdout in the format:

```
[ISO-timestamp] METHOD /path STATUS - Xms
```

Server-side errors (5xx) are additionally logged to stderr with a full stack trace in development mode.
